## 15) Чеклисты готовности

### 15.1 Definition of Ready (DoR) — можно брать в разработку
- [ ] README актуален, статус `approved`
- [ ] Требования (FR/NFR) согласованы
- [ ] Acceptance Criteria согласованы
- [ ] Дизайн готов и принят фронтендом
- [ ] API спек/контракты готовы и согласованы
- [ ] ML-интеграция понятна (если применимо)
- [ ] Оценка и декомпозиция есть (с учетом QA +30%)
- [ ] Риски/зависимости зафиксированы

### 15.2 Definition of Done (DoD) — фича “полностью готова”
- [ ] Код в prod
- [ ] QA пройден
- [ ] Приёмка на stage пройдена
- [ ] README обновлен под фактическую реализацию
- [ ] CHANGELOG обновлен
- [ ] Документация (внутренняя/пользовательская) обновлена при необходимости
- [ ] Мониторинг/метрики/алёрты настроены (если нужно)
