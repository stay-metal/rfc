## Feature Docs (docs/features)

Мы ведём документацию по продуктовым фичам в формате **docs-as-code**: у каждой фичи есть своя папка с единым «живым» документом, историей изменений и (при необходимости) журналом решений.  
Это позволяет фиксировать согласованные требования и поддерживать их актуальность до и после релиза.

### Структура

```txt
/docs/features/
  FEAT-0001-short-slug/
    README.md          # master-doc фичи (единый живой документ)
    CHANGELOG.md       # история изменений по фиче (итерации/релизы)
    decisions.md       # (опционально) журнал решений/компромиссов
    assets/            # (опционально) схемы/картинки/диаграммы
